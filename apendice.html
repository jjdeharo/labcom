Especificación matemática

Apéndice matemático: mecanismo bayesiano, IRT y entropía de Shannon

v1.0

Este apéndice describe con más detalle el mecanismo matemático del sistema adaptativo: cómo representa el nivel del alumno, cómo actualiza sus creencias con Bayes, cómo usa una función tipo IRT para relacionar nivel y dificultad, cómo mide la incertidumbre con entropía de Shannon y cómo elige la siguiente pregunta.

1. Representación del nivel como distribución de probabilidad

Para cada tipo de problema, el sistema no guarda un único valor de nivel, sino una distribución de probabilidad sobre tres hipótesis:

\[\Theta=\{-2,0,2\}\]

donde \(\theta=-2\) representa «no aprendido», \(\theta=0\) representa «aprendiendo» y \(\theta=2\) representa «dominado».

Para cada tipo \(t\), el estado del alumno es:

\[P_t(\theta)=\bigl(P_t(-2),\ P_t(0),\ P_t(2)\bigr)\]

Al inicio, como no hay información previa:

\[P_t(-2)=P_t(0)=P_t(2)=\frac13\]

Esto se hace de forma independiente para cada tipo de problema. Si los tipos son:

\[T=\{\text{perm},\ \text{permRep},\ \text{var},\ \text{varRep},\ \text{comb},\ \text{combRep}\}\]

entonces el sistema mantiene seis distribuciones separadas. El alumno no tiene «un nivel global» único: tiene una estimación independiente para cada tipo de problema.

2. Nivel esperado

A partir de la distribución de cada tipo, el sistema puede calcular un valor medio de nivel:

\[E_t[\theta] = \sum_{\theta_i \in \{-2,0,2\}} P_t(\theta_i)\cdot \theta_i = P_t(-2)(-2)+P_t(0)(0)+P_t(2)(2)\]

Ejemplo: si \(P_t(\theta)=(0{,}20,\ 0{,}30,\ 0{,}50)\), entonces:

\[E_t[\theta] = 0{,}20(-2)+0{,}30(0)+0{,}50(2) = -0{,}40+0+1 = 0{,}60\]

Ese valor, 0,60, resume la distribución, pero no la sustituye completamente. La información más rica sigue estando en la distribución completa.

3. Probabilidad de acierto mediante una función tipo IRT

Para actualizar el modelo, el sistema necesita estimar la probabilidad de que un alumno con cierto nivel \(\theta\) acierte un ejercicio de dificultad \(b\). Usa una función logística con suelo de azar:

\[P(A\mid \theta,b) = c+\frac{1-c}{1+e^{-a(\theta-b)}}\]

donde \(A\) significa acierto, \(a\) es el parámetro de discriminación (\(a=1{,}5\)) y \(c\) es el suelo de azar.

La dificultad de los ejercicios toma valores \(b=-1\) (fácil), \(b=0\) (medio) o \(b=1\) (difícil).

4. Interpretación de la función IRT

La cantidad decisiva es \(\theta-b\). Si \(\theta>b\), el nivel hipotético está por encima de la dificultad y la probabilidad de acierto es alta. Si \(\theta=b\), la logística básica vale \(1/2\), por lo que \(P(A\mid\theta=b)=c+\frac{1-c}{2}\). Si \(\theta<b\), la probabilidad baja, aunque no baja de \(c\).

Hipótesis\(\theta\)Relación con \(b=0\)Interpretación
No aprendido−2\(\theta<b\)Acierto poco probable
Aprendiendo0\(\theta=b\)Acierto intermedio
Dominado2\(\theta>b\)Acierto muy probable

Por eso un acierto en una pregunta difícil desplaza más la creencia hacia «dominado» que un acierto en una pregunta fácil.

6. Puntuación parcial del ejercicio

Cada ejercicio produce una puntuación parcial \(s\in[0,1]\), obtenida ponderando siete señales:

SeñalPeso
Orden correcto \((x_{\text{orden}})\)0,10
Alcance correcto \((x_{\text{alcance}})\)0,10
Repetición correcta \((x_{\text{rep}})\)0,10
Tipo correcto \((x_{\text{tipo}})\)0,25
\(n\) correcto \((x_n)\)0,15
\(k\) correcto \((x_k)\)0,15
Fórmula correcta \((x_{\text{formula}})\)0,15
\[s = 0{,}10x_{\text{orden}} + 0{,}10x_{\text{alcance}} + 0{,}10x_{\text{rep}} + 0{,}25x_{\text{tipo}} + 0{,}15x_n + 0{,}15x_k + 0{,}15x_{\text{formula}}\]

Cada variable vale 1 si la respuesta es correcta y 0 si es incorrecta. La suma de los pesos es 1, por tanto \(0\leq s\leq1\).

7. Verosimilitud suave a partir del score

El resultado del ejercicio no es solo acierto o fallo, sino una puntuación parcial \(s\). Por eso el sistema usa una verosimilitud interpolada:

\[L(\theta) = s\cdot P(A\mid \theta,b) + (1-s)\cdot P(F\mid \theta,b)\]

Como \(P(F\mid\theta,b)=1-P(A\mid\theta,b)\), si llamamos \(p_\theta=P(A\mid\theta,b)\):

\[L(\theta) = s\,p_\theta + (1-s)(1-p_\theta)\]

Casos importantes:

  • Si \(s=1\): \(L(\theta)=P(A\mid\theta,b)\) — acierto completo.
  • Si \(s=0\): \(L(\theta)=P(F\mid\theta,b)\) — fallo completo.
  • Si \(s=0{,}5\): \(L(\theta)=0{,}5\) para cualquier \(\theta\) — la actualización bayesiana es prácticamente nula.

8. Actualización bayesiana

Antes del ejercicio, el estado del tipo \(t\) es \(P_t^{\text{old}}(\theta_i)\). Para cada hipótesis se calcula la verosimilitud \(L(\theta_i)\), después el valor no normalizado \(q_i=P_t^{\text{old}}(\theta_i)\cdot L(\theta_i)\), y finalmente se normaliza:

\[P_t^{\text{new}}(\theta_i) = \frac{P_t^{\text{old}}(\theta_i)\cdot L(\theta_i)}{\sum_j P_t^{\text{old}}(\theta_j)\cdot L(\theta_j)}\]

Esta es la regla de Bayes aplicada a las tres hipótesis discretas.

Olvido exponencial en la fase de refuerzo

La regla anterior da el mismo peso a todas las respuestas, la primera y la última. Durante el diagnóstico inicial eso es lo correcto. Pero en la fase de refuerzo el alumno aprende mientras practica —y la creencia se conserva entre sesiones—, de modo que la evidencia antigua puede anclar la estimación en un estado ya superado. Por eso, en la fase de refuerzo el programa atenúa la creencia acumulada antes de cada actualización, elevándola a una potencia \(\lambda=0{,}95\) y renormalizando:

\[P_t^{\text{old}}(\theta_i) \;\leftarrow\; \frac{\left[P_t^{\text{old}}(\theta_i)\right]^{\lambda}}{\sum_j \left[P_t^{\text{old}}(\theta_j)\right]^{\lambda}}\]

Así, la respuesta de hace \(k\) ejercicios pesa \(\lambda^k\) y el sistema recuerda de forma efectiva las últimas \(1/(1-\lambda)=20\) respuestas: la estimación puede seguir al alumno cuando mejora. Durante la fase diagnóstica se usa \(\lambda=1\) (Bayes puro), para no distorsionar el informe inicial ni el ajuste del patrón de respuestas.

9. Ejemplo completo de actualización

Ejercicio de dificultad media (\(b=0\)) con distribución inicial uniforme \(P^{\text{old}}=(1/3,1/3,1/3)\). Probabilidades de acierto IRT aproximadas: \(P(A\mid -2,0)\approx0{,}40\), \(P(A\mid 0,0)\approx0{,}68\), \(P(A\mid 2,0)\approx0{,}97\).

Caso 1: acierto completo (\(s=1\))

Las verosimilitudes son \(L(-2)=0{,}40\), \(L(0)=0{,}68\), \(L(2)=0{,}97\). Posterior:

\[P^{\text{new}}(-2)\approx0{,}195\quad P^{\text{new}}(0)\approx0{,}332\quad P^{\text{new}}(2)\approx0{,}473\]

La hipótesis «dominado» gana peso.

Caso 2: fallo completo (\(s=0\))

Las verosimilitudes son las de fallo: \(L(-2)=0{,}60\), \(L(0)=0{,}32\), \(L(2)=0{,}03\). Posterior:

\[P^{\text{new}}(-2)\approx0{,}632\quad P^{\text{new}}(0)\approx0{,}337\quad P^{\text{new}}(2)\approx0{,}032\]

La hipótesis «no aprendido» gana peso.

Caso 3: resultado parcial (\(s=0{,}75\))

\[L(-2)=0{,}75(0{,}40)+0{,}25(0{,}60)=0{,}45\] \[L(0)=0{,}75(0{,}68)+0{,}25(0{,}32)=0{,}59\] \[L(2)=0{,}75(0{,}97)+0{,}25(0{,}03)=0{,}735\]

El posterior se desplaza hacia «dominado», pero menos que con \(s=1\).

10. Estimación visible de éxito

El porcentaje mostrado al alumno es la probabilidad esperada de resolver correctamente un ejercicio medio:

\[\widehat{P}_t(A) = \sum_i P_t(\theta_i)\cdot P(A\mid \theta_i,b=0)\]

Ejemplo con \(P_t(\theta)=(0{,}20,\ 0{,}30,\ 0{,}50)\):

\[\widehat{P}_t(A) = 0{,}20(0{,}40)+0{,}30(0{,}68)+0{,}50(0{,}97) = 0{,}08+0{,}204+0{,}485 = 0{,}769 \approx 77\%\]

11. Entropía de Shannon

La incertidumbre de una distribución se mide con entropía de Shannon:

\[H(P) = -\sum_i p_i\log_2(p_i)\]

Para una distribución uniforme \((1/3,1/3,1/3)\), la entropía es máxima: \(H=\log_2(3)\). Para una distribución concentrada como \((0{,}05,\ 0{,}10,\ 0{,}85)\), la entropía es mucho menor. El sistema usa la entropía normalizada:

\[H_{\text{norm}}(P) = \frac{H(P)}{\log_2(3)} \in [0,1]\]

Si \(H_{\text{norm}}=1\), la incertidumbre es máxima. Si \(H_{\text{norm}}\approx0\), el sistema está muy seguro.

12. Ganancia de información esperada

Para elegir un ejercicio, el sistema calcula cuánta incertidumbre espera reducir:

\[IG = H(P) - \bigl[P(A)\,H(P_{\text{post},A}) + P(F)\,H(P_{\text{post},F})\bigr]\]

donde:

\[P(A) = \sum_i P(\theta_i)P(A\mid\theta_i,b)\qquad P(F)=1-P(A)\] \[P_{\text{post},A}(\theta_i) = \frac{P(\theta_i)P(A\mid\theta_i,b)}{P(A)}\qquad P_{\text{post},F}(\theta_i) = \frac{P(\theta_i)P(F\mid\theta_i,b)}{P(F)}\]

Un ejercicio tiene alta ganancia de información si, al responderlo, reduce mucho la incertidumbre sobre la distribución del alumno.

13. Intuición de la ganancia de información

Un ejercicio muy fácil suele tener poca ganancia: si el alumno acierta, aprendemos poco porque el acierto era esperable. Un ejercicio muy difícil también puede tener poca ganancia: si el alumno falla, aprendemos poco porque el fallo era esperable.

Los ejercicios más informativos suelen ser aquellos cuya dificultad se encuentra cerca del nivel estimado del alumno, porque ahí tanto el acierto como el fallo son posibles y ambos escenarios ayudan a diferenciar hipótesis.

14. Ajuste de dificultad

En la fase de refuerzo, el sistema también valora si la dificultad del ejercicio encaja con el nivel estimado del alumno. Primero calcula el nivel esperado \(\bar{\theta}=E[\theta]\), después el ajuste:

\[Fit(b,\bar{\theta}) = e^{-0{,}5(b-\bar{\theta})^2}\]

Si \(b=\bar{\theta}\), el ajuste es máximo (\(Fit=1\)). Si \(b\) se aleja de \(\bar{\theta}\), el ajuste disminuye:

\(|b-\bar{\theta}|\)\(Fit\)

2\(e^{-2}\approx0{,}135\)

15. Puntuación de un ejercicio candidato

Durante la fase diagnóstica, el sistema prioriza la ganancia de información:

\[Score_{\text{diagnóstico}}(p)=IG(p)\]

Durante la fase de refuerzo, combina ganancia de información y ajuste de dificultad:

\[Score_{\text{refuerzo}}(p) = 0{,}65\cdot IG_{\text{norm}}(p) + 0{,}35\cdot Fit(p)\]

donde \(IG_{\text{norm}}(p)=IG(p)/IG_{\max}\) normaliza la ganancia respecto al mejor candidato disponible.

16. Selección del tipo de problema

Fase diagnóstica

El sistema prioriza los tipos con mayor valor de diagnóstico:

\[D_t = missing_t + H_{\text{norm}}(P_t)\]

donde \(missing_t = \max(0, 2-n_t)\) es el número de intentos que faltan hasta el mínimo de dos. Esto combina falta de muestra e incertidumbre sobre el nivel.

Fase de refuerzo

Cuando todos los tipos tienen suficientes intentos, el sistema prioriza los tipos más débiles:

\[W_t = 1-\widehat{P}_t(A)\]

El sistema elige entre los tipos con mayor debilidad, introduciendo cierta aleatoriedad entre candidatos cercanos para evitar repetición excesiva.

17. Diagnóstico por habilidades transversales

Además de las distribuciones por tipo, el sistema mantiene distribuciones bayesianas para seis habilidades transversales:

\[D = \{\text{orden},\ \text{alcance},\ \text{repetición},\ \text{tipo},\ \text{nk},\ \text{fórmula}\}\]

Para cada habilidad \(d\), hay una distribución \(P_d(\theta)\) que se actualiza con una señal binaria \(r_d\in\{0,1\}\). Cada habilidad tiene su propio suelo de azar:

HabilidadSuelo de azar
Orden\(1/2\)
Alcance\(1/2\)
Repetición\(1/2\)
Tipo\(1/6\)
\(n,k\)\(1/2\)
Fórmula\(1/6\)
\[P(A_d\mid\theta,b) = c_d + \frac{1-c_d}{1+e^{-a(\theta-b)}}\]

18. Por qué los porcentajes de habilidades no son directamente comparables

El suelo de azar de «orden» es \(1/2\) y el de «tipo» es \(1/6\). Esto significa que el mismo nivel \(\theta\) produce porcentajes visibles diferentes según la habilidad: en una pregunta sí/no el suelo ya es 50 %, mientras que elegir entre seis tipos tiene un suelo de ~16,7 %.

Por tanto, un 75 % en orden no tiene exactamente el mismo significado que un 75 % en tipo. El porcentaje depende del suelo de azar. El nivel latente \(\theta\) es la referencia común.

19. Guardado del diagnóstico inicial

Cuando todos los tipos tienen al menos dos ejercicios realizados, el sistema guarda una copia del estado:

\[Snapshot = \{P_t(\theta)\}_{t\in T}\]

Ese snapshot permite comparar el nivel al terminar el diagnóstico con el nivel actual después de la fase de refuerzo:

\[\Delta_t = \widehat{P}_t^{\text{actual}}(A) - \widehat{P}_t^{\text{diagnóstico}}(A)\]

Si \(\Delta_t>0\) hay mejora estimada; si \(\Delta_t<0\) hay descenso; si \(\Delta_t\approx0\) el nivel se mantiene estable.

20. Resumen formal del ciclo completo

Paso 1: estado previo

Para cada tipo: \(P_t^{\text{old}}(\theta)\). Para cada habilidad: \(P_d^{\text{old}}(\theta)\).

Paso 2: selección del tipo

En diagnóstico: \(t^*=\arg\max_t\bigl(missing_t + H_{\text{norm}}(P_t)\bigr)\). En refuerzo: \(t^*=\arg\max_t\bigl(1-\widehat{P}_t(A)\bigr)\), con selección aleatoria entre candidatos cercanos al máximo.

Paso 3: selección del ejercicio

En diagnóstico: \(p^*=\arg\max_p IG(p)\). En refuerzo: \(p^*=\arg\max_p\bigl(0{,}65\,IG_{\text{norm}}(p)+0{,}35\,Fit(p)\bigr)\).

Paso 4: respuesta del alumno

El alumno produce señales: \(x_{\text{orden}}, x_{\text{alcance}}, x_{\text{rep}}, x_{\text{tipo}}, x_n, x_k, x_{\text{formula}}\).

Paso 5: cálculo del score

\[s = 0{,}10x_{\text{orden}} + 0{,}10x_{\text{alcance}} + 0{,}10x_{\text{rep}} + 0{,}25x_{\text{tipo}} + 0{,}15x_n + 0{,}15x_k + 0{,}15x_{\text{formula}}\]

Paso 6: actualización del tipo

\[P_{t^*}^{\text{new}}(\theta_i) = \frac{P_{t^*}^{\text{old}}(\theta_i)\,L(\theta_i)}{\sum_j P_{t^*}^{\text{old}}(\theta_j)\,L(\theta_j)} \qquad L(\theta_i)=s\,P(A\mid\theta_i,b)+(1-s)\,P(F\mid\theta_i,b)\]

Paso 7: actualización de habilidades

Para cada habilidad \(d\): misma regla bayesiana con \(L_d\) usando el resultado binario de esa habilidad y su suelo de azar \(c_d\).

Paso 8: estimación visible

\[\widehat{P}_t(A) = \sum_i P_t(\theta_i)P(A\mid\theta_i,0) \qquad E_t[\theta] = \sum_i P_t(\theta_i)\,\theta_i\]

Con estos valores se generan barras, semáforo e informe.

21. Idea matemática central

\[\boxed{\text{Bayes} + \text{IRT} + \text{Shannon} + \text{score parcial}}\]

Bayes actualiza la creencia: \(P(\theta\mid R)\). IRT estima la probabilidad de acierto según nivel y dificultad: \(P(A\mid\theta,b)\). Shannon mide la incertidumbre: \(H(P)\). La ganancia de información decide qué pregunta reduce más esa incertidumbre: \(IG=H_{\text{antes}}-E[H_{\text{después}}]\). El score parcial transforma una resolución compleja en una señal gradual: \(s\in[0,1]\).

El resultado es un sistema en el que cada respuesta modifica el estado interno del alumno, y ese nuevo estado modifica la siguiente pregunta.

Detalles de implementación

22. Detalles de implementación

Las secciones anteriores describen el mecanismo matemático. Esta sección recoge las decisiones concretas de la implementación que afectan a lo que el alumno ve, pero que no forman parte del núcleo bayesiano.

22.1 Topes del porcentaje visible según el número de intentos

La estimación cruda \(\widehat{P}_t(A)\) no se muestra siempre directamente. Para evitar conclusiones excesivamente optimistas con poca evidencia, el sistema limita el porcentaje visible según el número de intentos \(n_t\). Los umbrales se derivan de la función IRT con dificultad media:

\[\text{LOW\_MAX} = \operatorname{round}\bigl(100\cdot P(A\mid\theta=-0{,}3,\,b=0)\bigr) = 61\] \[\text{HIGH\_MIN} = \operatorname{round}\bigl(100\cdot P(A\mid\theta=1{,}4,\,b=0)\bigr) = 93\]

El porcentaje mostrado es:

\[\widehat{P}_t^{\text{visible}} = \begin{cases} \min\bigl(\widehat{P}_t(A),\ 60\bigr) & \text{si } n_t < 2 \\ \min\bigl(\widehat{P}_t(A),\ 92\bigr) & \text{si } 2 \le n_t < 4 \\ \widehat{P}_t(A) & \text{si } n_t \ge 4 \end{cases}\]

22.2 Umbrales del semáforo

El color del semáforo depende del nivel esperado \(\bar{\theta}_t\) y del número de intentos \(n_t\):

CondiciónColor
\(n_t=0\)Gris (sin datos)
\(n_t < 2\)Rojo
\(2\le n_t < 4\) y \(\bar{\theta}_t\ge-0{,}3\)Amarillo
\(2\le n_t < 4\) y \(\bar{\theta}_t<-0{,}3\)Rojo
\(n_t\ge4\) y \(\bar{\theta}_t\ge1{,}4\)Verde
\(n_t\ge4\) y \(-0{,}3\le\bar{\theta}_t<1{,}4\)Amarillo
\(n_t\ge4\) y \(\bar{\theta}_t<-0{,}3\)Rojo

Los cortes \(\bar{\theta}=-0{,}3\) y \(\bar{\theta}=1{,}4\) son los mismos que generan las bandas LOW_MAX y HIGH_MIN de la sección 22.1, de modo que color y porcentaje son coherentes entre sí.

22.3 Aleatoriedad en la selección

Un candidato \(p\) entra en el conjunto de candidatos elegibles si \(Score(p)\ge Score_{\max}-\tau\), con tolerancias distintas según el contexto:

22.4 Generación de ejercicios y asignación de dificultad

Cuando se agotan los ejercicios disponibles de un tipo en el banco, el sistema genera ejercicios nuevos de forma procedimental. La dificultad \(b\in\{-1,0,1\}\) se asigna mediante reglas basadas en el tamaño de \(n\) y \(k\) (o del resultado numérico, según el tipo). Estas reglas son una heurística de diseño, no una calibración psicométrica.

22.5 Diagnóstico de \(n\) y \(k\) como una sola habilidad

En el cálculo del score, \(n\) y \(k\) son dos señales independientes con peso 0,15 cada una. En cambio, en el diagnóstico por habilidades transversales ambas se agrupan en la habilidad \(nk\), que se considera acertada solo si \(n\) y \(k\) son correctos a la vez:

\[r_{nk} = x_n\cdot x_k\]

El score puede otorgar crédito parcial por acertar solo uno de los dos, pero la habilidad transversal \(nk\) exige acertar ambos.

23. Validación y fiabilidad del diagnóstico (opcional)

Las comprobaciones de esta sección no forman parte del flujo obligatorio: son una capa opcional que aumenta la honestidad del diagnóstico sin necesidad de datos empíricos.

23.1 Ajuste del patrón individual (person-fit \(l_z\))

Al cerrar el diagnóstico inicial, el sistema evalúa si el patrón de respuestas es coherente con el nivel estimado. Para cada pregunta respondida durante el diagnóstico, con dificultad \(b_i\) y puntuación observada \(s_i\in[0,1]\), se toma \(\widehat{\theta}\) como la hipótesis más probable (MAP) del tipo correspondiente y \(P_i = P(A\mid\widehat{\theta},\,b_i)\). El índice estandarizado de Drasgow es:

\[l_z = \frac{l_0 - E[l_0]}{\sqrt{\operatorname{Var}[l_0]}}\]

con:

\[l_0 = \sum_i \bigl[s_i\ln P_i + (1-s_i)\ln(1-P_i)\bigr]\] \[E[l_0] = \sum_i \bigl[P_i\ln P_i + (1-P_i)\ln(1-P_i)\bigr]\] \[\operatorname{Var}[l_0] = \sum_i P_i(1-P_i)\left[\ln\frac{P_i}{1-P_i}\right]^2\]

Un valor \(l_z\approx0\) indica buen ajuste. Un valor muy negativo (orientativamente \(l_z<-2\)) señala respuestas incoherentes con la dificultad —aciertos en lo difícil y fallos en lo fácil, o azar—, y el sistema marca el diagnóstico como provisional en el informe. Es una señal de cautela, no una prueba formal; con pocas preguntas es solo orientativo, por lo que se calcula únicamente a partir de tres respuestas.

23.2 Separabilidad del diseño (Monte Carlo)

Como propiedad del test (no del alumno), se puede estimar con qué fiabilidad el banco distingue los niveles. Se generan respondentes sintéticos situados en el \(\theta\) de cada hipótesis, se les pasa el test reutilizando la misma selección adaptativa y se construye la matriz de confusión (nivel real frente a nivel diagnosticado). La diagonal recoge los aciertos del diseño y su media es la exactitud.

Hay que presentarla como fiabilidad bajo el modelo, nunca como validez empírica: los respondentes salen del propio modelo, así que mide si el diseño discrimina los niveles, no si los parámetros reflejan la realidad. Es una herramienta del creador del recurso, no del alumnado, y no se muestra en la interfaz del alumno: vive en una utilidad aparte (validacion-banco.html) que el autor puede ejecutar al diseñar o revisar el banco.
