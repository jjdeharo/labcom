Conceptual explanation

Bayesian functioning of the adaptive practice system

This program does not merely count correct and incorrect answers. Its functioning is based on a formative adaptive model that estimates, probabilistically, which types of problems a student has mastered, which they are learning, and which need reinforcement. To do this it combines three main ideas: Bayesian inference, an IRT-type function to relate level and difficulty, and Shannon entropy to select informative questions.

It does not aim to produce an exact psychometric score, but rather to guide practice: deciding what to ask next, interpreting errors and offering a useful diagnosis.

1. The program does not store a grade: it stores a Bayesian belief

The first thing to understand is that the system does not calculate the student's level as a simple percentage of correct answers. It does not work by saying: "they got 7 out of 10, so they know 70%". It does something different: it maintains a probabilistic belief about the student's state.

For each problem type, the program considers three possible hypotheses:

Hypothesisθ valueMeaning
Not learned−2The student has not yet mastered this type
Learning0The student is in progress
Mastered+2The student solves it with ease

At the start, since the program knows nothing about the student, it distributes probability equally:

\[P(\text{not learned})=\frac13\] \[P(\text{learning})=\frac13\] \[P(\text{mastered})=\frac13\]

That is, for each type it starts as:

\[[1/3,\ 1/3,\ 1/3]\]

That does not mean the student has a 33% grade. It means that, with the available initial information, all three hypotheses are equally possible.

Additionally, this belief is stored separately for each problem type. A student may be very advanced in variations with repetition while having difficulties with combinations. The program does not describe the student "as a whole", but competency by competency.

This distribution is then summarised by a mean level:

\[\theta_{\text{estimated}} = P(\text{not learned})\cdot(-2) + P(\text{learning})\cdot0 + P(\text{mastered})\cdot2\]

For example, if after several exercises the belief about combinations were:

\[[0.20,\ 0.30,\ 0.50]\]

the estimated level would be:

\[\theta = 0.20\cdot(-2)+0.30\cdot0+0.50\cdot2=0.6\]

That 0.6 is not a grade. It is an approximate position on an internal mastery scale.

The fundamental idea is that "level" is not a counter of correct answers, but a probability distribution over three states: not learned, learning and mastered.

2. Bayes comes in when the student answers

Each time the student solves an exercise, the program updates its belief using Bayes' rule.

The structure is:

\[P(H_i \mid R) = \frac{P(H_i)\cdot P(R \mid H_i)}{\sum_j P(H_j)\cdot P(R \mid H_j)}\]

Where:

  • \(H_i\) is a hypothesis: not learned, learning or mastered.
  • \(R\) is the observed result.
  • \(P(H_i)\) is the belief before the exercise.
  • \(P(R \mid H_i)\) is the probability of obtaining that result if that hypothesis were true.

In simple terms, the program asks itself:

If the student were really at the "not learned" level, what probability would there be that they answered this way?
If they were at "learning", what probability would there be?
If they were at "mastered", what probability would there be?

It then compares those probabilities and shifts the belief towards the hypothesis that best explains the response.

If the student answers an exercise well, the "mastered" hypothesis gains weight. If they fail, "not learned" gains weight. If the answer is intermediate, the update is softer.

This point is essential: Bayes does not automatically convert correct answers into percentages. What it does is move probability between hypotheses according to how compatible the student's response was with each one.

3. The IRT part: student level vs exercise difficulty

To apply Bayes, the program needs to estimate the probability that a student of a certain level would answer an exercise of a certain difficulty correctly. This is where the IRT-type part comes in.

The program uses a logistic function:

\[P(\text{correct}) = c + \frac{1-c}{1+e^{-a(\theta-b)}}\]

Where:

  • \(\theta\) is the hypothetical student level.
  • \(b\) is the exercise difficulty.
  • \(a\) controls the discrimination of the question.
  • \(c\) is the guessing floor.

The idea is simple: if \(\theta > b\), the student's level is above the exercise difficulty and a correct answer is quite likely. If \(\theta < b\), the difficulty exceeds the estimated level and a correct answer is less likely.

Exercises have an approximate difficulty:

DifficultyInterpretation
−1Easy
0Medium
+1Hard

So, getting a hard question right provides more positive information than getting an easy one right, because the correct answer was less expected. Similarly, getting an easy question wrong is more significant than getting a hard one wrong.

If a student gets a medium-difficulty exercise right, the program compares that result with the three hypotheses \(\theta=-2\), \(\theta=0\) and \(\theta=2\). If the correct answer is much more compatible with \(\theta=2\) than with \(\theta=-2\), the probability of "mastered" increases.

Given this hypothetical level and this difficulty, what was the probability of getting it right? — Bayes uses this answer to update the belief.

4. Before applying Bayes, the program calculates a partial score

The program does not evaluate an exercise simply as "right" or "wrong". It divides it into several parts, because solving a combinatorics problem involves several steps.

The student must: (1) decide whether order matters; (2) decide whether all elements are used or only some; (3) decide whether repetition is allowed; (4) name the problem type; (5) identify \(n\); (6) identify \(k\); (7) choose the formula.

Each part has a weight:

Part evaluatedWeight
Order10%
Scope: all or some10%
Repetition10%
Problem type25%
Identification of \(n\)15%
Identification of \(k\)15%
Formula15%
\[\text{score} = 0.10\cdot\text{order} + 0.10\cdot\text{scope} + 0.10\cdot\text{rep} + 0.25\cdot\text{type} + 0.15\cdot n + 0.15\cdot k + 0.15\cdot\text{formula}\]

Each part equals 1 if correct and 0 if incorrect. For example, if the student gets order, repetition, type, \(n\) and formula right, but fails scope and \(k\):

\[0.10 + 0.10 + 0.25 + 0.15 + 0.15 = 0.75\]

This is important because the system does not only reward the final result. What matters is the process: reading the problem, classifying, identifying \(n\) and \(k\), and choosing the formula.

5. What a score of 1, 0.75, 0.5, 0.25 or 0 does

The score controls the direction and intensity of the Bayesian update. If the score is 1, the exercise is interpreted as a complete correct answer; if 0, as a complete failure. Intermediate values produce softer updates.

To connect the score with Bayes, the program needs to know how well each hypothesis is supported by the result. That measure is called the likelihood, \(L(\theta)\): it indicates, for each hypothetical value of θ, how compatible the observed result is with it. It is the \(P(R\mid H_i)\) term in Bayes' rule, but expressed from the score rather than a binary correct/incorrect signal.

The program calculates this soft likelihood as follows:

\[L(\theta)=\text{score}\cdot P(A\mid\theta)+(1-\text{score})\cdot P(F\mid\theta)\]

where \(P(F\mid\theta)=1-P(A\mid\theta)\). This allows partially correct exercises to be treated gradually.

If the score is 0.5, the likelihood is practically the same for all hypotheses: the system has no reason to change the distribution.

Above 0.5, the result pushes towards "mastered".
Below 0.5, it pushes towards "not learned".
Around 0.5, it barely informs.

6. How it converts belief into percentage and traffic light

After updating Bayes, the program has a distribution for each type. The student does not see this distribution directly: the program converts it into an estimated percentage, a bar and a traffic light.

The estimated probability of success on a medium-difficulty exercise is:

\[P(\text{correct}) = \sum_i P(H_i)\cdot P(\text{correct}\mid \theta_i,\ b=0)\]
With the current belief, what probability would this student have of solving a medium exercise of this type correctly?

The traffic light translates the level into categories: red (low mastery), yellow (in progress) and green (mastered). Additionally, the program avoids declaring "mastered" too soon: it needs a minimum number of attempts before it can show high mastery. Thus the visible panel is a pedagogical translation of the internal Bayesian belief.

7. How it chooses which problem type to ask

The adaptive system works in two phases: diagnosis and reinforcement.

Diagnostic phase

At the start, the program tries to obtain a minimum sample of all types. Since there are six problem types and it requires two exercises per type, the initial diagnosis consists of \(6 \times 2 = 12\) exercises.

Uncertainty is measured with Shannon entropy:

\[H(p)=-\sum p_i\log_2(p_i)\]

A uniform distribution \([1/3,\ 1/3,\ 1/3]\) has maximum uncertainty; a concentrated distribution \([0.05,\ 0.10,\ 0.85]\) has low uncertainty. During diagnosis, the system prioritises under-practised types and those with the most uncertainty.

Reinforcement phase

When all types have at least two exercises, the diagnostic phase ends. The program then changes its criterion:

Which type does the student have the lowest estimated probability of success in?

The weakness is \(1-P(\text{success})\). If the system estimates that the student has a 45% probability of success in combinations: \(1-0.45=0.55\). If in permutations 85%: \(1-0.85=0.15\). Combinations have higher priority. The program does not always choose the type with the worst score: it selects randomly from the best candidates to avoid rigid, repetitive practice.

8. How it selects the specific exercise within the chosen type

Once the type is chosen, the system calculates the expected information gain for each candidate exercise:

If I give this exercise, how much do I expect to reduce my uncertainty about the student's level?
\[IG = H(\text{before}) - \bigl[P(A)\cdot H(\text{after correct}) + P(F)\cdot H(\text{after wrong})\bigr]\]

A good exercise is one that, whatever the outcome, helps clarify the student's level. This is why neither the easiest nor the hardest questions are always best: the most informative are those close to the student's level, where the outcome is still uncertain.

During diagnosis, the program prioritises almost exclusively information gain. During reinforcement, it combines two criteria: 65% information gain and 35% difficulty fit. Difficulty fit measures whether the exercise difficulty is close to the student's estimated level.

9. The skills diagnosis: knowing where the student fails

In addition to the level by problem type, the program maintains a diagnosis by cross-cutting skills:

SkillMeaning
OrderKnowing whether order matters
ScopeKnowing whether all or only some elements are used
RepetitionKnowing whether repetition is allowed
TypeNaming the problem type
\(n,k\)Correctly identifying \(n\) and \(k\)
FormulaChoosing the correct formula

Each skill has its own Bayesian belief. This allows distinguishing between which type needs practice (level by type) and which part of the process fails (level by skill).

For example, two students may fail combinations for different reasons: one may correctly identify \(n\) and \(k\) but choose the wrong formula; another may fail from reading the problem because they cannot distinguish whether order matters. Both have difficulties with combinations, but they do not need the same help.

This diagnosis does not directly guide exercise selection, but is very useful for interpreting errors and directing teaching intervention.

10. How the report is generated

The report appears when the diagnostic phase is complete. The program saves a "snapshot" of the state at the end of the initial 12 exercises and compares it with the current level after the reinforcement phase:

TypeInitial diagnosisCurrent level
Combinations48%67%
Variations72%78%
Permutations90%91%

The report shows: level at the end of diagnosis, current level, progress by type, strengths, types that need practice, personalised recommendation and skills diagnosis (separating problem reading and resolution steps).

11. The full cycle

\[\boxed{\text{current state} \rightarrow \text{adaptive selection} \rightarrow \text{partial answer} \rightarrow \text{score} \rightarrow \text{Bayes} \rightarrow \text{new state} \rightarrow \text{next selection}}\]

The program is adaptive: it does not follow a fixed list of exercises, but each response modifies the next decision. If the student has failed combinations, combinations will gain priority; if they have done very well on variations, variations will lose urgency.

12. What parts are solid and what parts are heuristic

Solid aspects

The system does not use a simple grade: it maintains a probabilistic belief per problem type, which allows expressing uncertainty. It evaluates the process, not just the final result. It uses Shannon entropy to choose informative questions, not simply the most difficult ones. And it performs a diagnostic phase before reinforcing, avoiding too-rapid adaptations based on little data.

Heuristic aspects

The IRT function —the logistic curve \(P(A\mid\theta,b)\) that estimates the probability of a correct answer given the student's level \(\theta\) and exercise difficulty \(b\)— is heuristic: its parameters do not come from empirical calibration with real students. The question difficulty is approximate. The score weights are reasonable but debatable. The guessing floor simplifies: in practice, student errors are not completely random. The model treats types as fairly independent competencies, when in reality there are connections. And the three-level scale is clear but coarse.

Conclusion

The program combines:

\[\text{Bayes} + \text{heuristic IRT} + \text{Shannon entropy} + \text{partial process score}\]

Bayes updates the belief about the student. IRT estimates what probability of a correct answer corresponds to each level and difficulty. Shannon measures uncertainty and helps choose informative questions. The partial score allows evaluating the resolution process, not just the result.

The system does not simply say "you got this many exercises right", but something more useful: according to your responses, these types seem mastered, these are in progress, these need reinforcement, and these specific reasoning steps should be reviewed. Its main value lies in guiding practice and making the learning process visible.
