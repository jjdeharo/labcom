Especificación matemática

Apéndice matemático: mecanismo bayesiano, IRT e entropía de Shannon

Este apéndice describe con máis detalle o mecanismo matemático do sistema adaptativo: como representa o nivel do alumno, como actualiza as súas crenzas con Bayes, como usa unha función tipo IRT para relacionar nivel e dificultade, como mide a incerteza coa entropía de Shannon e como elixe a próxima pregunta.

1. Representación do nivel como distribución de probabilidade

Para cada tipo de problema, o sistema non garda un único valor de nivel, senón unha distribución de probabilidade sobre tres hipóteses:

\[\Theta=\{-2,0,2\}\]

onde \(\theta=-2\) representa «non aprendido», \(\theta=0\) representa «aprendendo» e \(\theta=2\) representa «dominado».

Para cada tipo \(t\), o estado do alumno é:

\[P_t(\theta)=\bigl(P_t(-2),\ P_t(0),\ P_t(2)\bigr)\]

Ao principio, sen información previa:

\[P_t(-2)=P_t(0)=P_t(2)=\frac13\]

Isto faise de forma independente para cada tipo de problema. Se os tipos son:

\[T=\{\text{perm},\ \text{permRep},\ \text{var},\ \text{varRep},\ \text{comb},\ \text{combRep}\}\]

o sistema mantén seis distribucións separadas. O alumno non ten «un único nivel global»: ten unha estimación independente para cada tipo de problema.

2. Nivel esperado

A partir da distribución para cada tipo, o sistema pode calcular un valor medio de nivel:

\[E_t[\theta] = \sum_{\theta_i \in \{-2,0,2\}} P_t(\theta_i)\cdot \theta_i = P_t(-2)(-2)+P_t(0)(0)+P_t(2)(2)\]

Exemplo: se \(P_t(\theta)=(0{,}20,\ 0{,}30,\ 0{,}50)\), entón:

\[E_t[\theta] = 0{,}20(-2)+0{,}30(0)+0{,}50(2) = -0{,}40+0+1 = 0{,}60\]

Este valor resume a distribución, pero non a substitúe completamente. A información máis rica segue sendo a distribución completa.

3. Probabilidade de acerto mediante unha función tipo IRT

Para actualizar o modelo, o sistema necesita estimar a probabilidade de que un alumno con certo nivel \(\theta\) acerte un exercicio de dificultade \(b\). Usa unha función loxística con chan de azar:

\[P(A\mid \theta,b) = c+\frac{1-c}{1+e^{-a(\theta-b)}}\]

onde \(A\) significa resposta correcta, \(a\) é o parámetro de discriminación (\(a=1{,}5\)) e \(c\) é o chan de azar.

As dificultades dos exercicios toman valores \(b=-1\) (fácil), \(b=0\) (media) ou \(b=1\) (difícil).

4. Interpretación da función IRT

A cantidade decisiva é \(\theta-b\). Se \(\theta>b\), o nivel hipotético está por riba da dificultade e a probabilidade de acerto é alta. Se \(\theta=b\), a loxística básica vale \(1/2\), de xeito que \(P(A\mid\theta=b)=c+\frac{1-c}{2}\). Se \(\theta<b\), a probabilidade baixa, pero non por baixo de \(c\).

Hipótese\(\theta\)Relación con \(b=0\)Interpretación
Non aprendido−2\(\theta<b\)Acerto pouco probable
Aprendendo0\(\theta=b\)Probabilidade intermedia
Dominado2\(\theta>b\)Acerto moi probable

Por iso, acertar unha pregunta difícil despraza a crenza cara a «dominado» máis que acertar unha fácil.

5. Chan de azar

O chan de azar para o dominio do tipo global calcúlase a partir do rendemento esperado por azar en todo o proceso:

PartePesoAzar aprox.

Tipo de problema25 %\(1/6\)
Identificación de \(n\) e \(k\)30 %\(1/2\)
Fórmula15 %\(1/6\)
\[c_{\text{tipo}} = 0{,}30\cdot\tfrac12 + 0{,}25\cdot\tfrac16 + 0{,}30\cdot\tfrac12 + 0{,}15\cdot\tfrac16 = 0{,}15+0{,}042+0{,}15+0{,}025 = \frac{11}{30} \approx 0{,}367\]

6. Puntuación parcial do exercicio

Cada exercicio produce unha puntuación parcial \(s\in[0,1]\), obtida ponderando sete sinais:

SinalPeso
Orde correcta \((x_{\text{orde}})\)0,10
Alcance correcto \((x_{\text{alcance}})\)0,10
Repetición correcta \((x_{\text{rep}})\)0,10
Tipo correcto \((x_{\text{tipo}})\)0,25
\(n\) correcta \((x_n)\)0,15
\(k\) correcta \((x_k)\)0,15
Fórmula correcta \((x_{\text{fórmula}})\)0,15
\[s = 0{,}10x_{\text{orde}} + 0{,}10x_{\text{alcance}} + 0{,}10x_{\text{rep}} + 0{,}25x_{\text{tipo}} + 0{,}15x_n + 0{,}15x_k + 0{,}15x_{\text{fórmula}}\]

Cada variable vale 1 se a resposta é correcta e 0 se é incorrecta. Os pesos suman 1, polo tanto \(0\leq s\leq1\).

7. Verosimilitude suave a partir da puntuación

O resultado dun exercicio non é simplemente correcto ou incorrecto, senón unha puntuación parcial \(s\). O sistema usa por tanto unha verosimilitude interpolada:

\[L(\theta) = s\cdot P(A\mid \theta,b) + (1-s)\cdot P(F\mid \theta,b)\]

Como \(P(F\mid\theta,b)=1-P(A\mid\theta,b)\), escribindo \(p_\theta=P(A\mid\theta,b)\):

\[L(\theta) = s\,p_\theta + (1-s)(1-p_\theta)\]

Casos especiais:

  • Se \(s=1\): \(L(\theta)=P(A\mid\theta,b)\) — acerto completo.
  • Se \(s=0\): \(L(\theta)=P(F\mid\theta,b)\) — fallo completo.
  • Se \(s=0{,}5\): \(L(\theta)=0{,}5\) para calquera \(\theta\) — a actualización bayesiana é practicamente nula.

8. Actualización bayesiana

Antes do exercicio, o estado do tipo \(t\) é \(P_t^{\text{antigo}}(\theta_i)\). Para cada hipótese calcúlase a verosimilitude \(L(\theta_i)\), despois o valor non normalizado \(q_i=P_t^{\text{antigo}}(\theta_i)\cdot L(\theta_i)\), e finalmente normalízase:

\[P_t^{\text{novo}}(\theta_i) = \frac{P_t^{\text{antigo}}(\theta_i)\cdot L(\theta_i)}{\sum_j P_t^{\text{antigo}}(\theta_j)\cdot L(\theta_j)}\]

Isto é a regra de Bayes aplicada ás tres hipóteses discretas.

9. Exemplo completo de actualización

Exercicio de dificultade media (\(b=0\)) con distribución inicial uniforme \(P^{\text{antigo}}=(1/3,1/3,1/3)\). Probabilidades de acerto IRT aproximadas: \(P(A\mid -2,0)\approx0{,}40\), \(P(A\mid 0,0)\approx0{,}68\), \(P(A\mid 2,0)\approx0{,}97\).

Caso 1: acerto completo (\(s=1\))

Verosimilitudes: \(L(-2)=0{,}40\), \(L(0)=0{,}68\), \(L(2)=0{,}97\). Posterior:

\[P^{\text{novo}}(-2)\approx0{,}195\quad P^{\text{novo}}(0)\approx0{,}332\quad P^{\text{novo}}(2)\approx0{,}473\]

A hipótese «dominado» gaña peso.

Caso 2: fallo completo (\(s=0\))

Verosimilitudes de fallo: \(L(-2)=0{,}60\), \(L(0)=0{,}32\), \(L(2)=0{,}03\). Posterior:

\[P^{\text{novo}}(-2)\approx0{,}632\quad P^{\text{novo}}(0)\approx0{,}337\quad P^{\text{novo}}(2)\approx0{,}032\]

A hipótese «non aprendido» gaña peso.

Caso 3: resultado parcial (\(s=0{,}75\))

\[L(-2)=0{,}75(0{,}40)+0{,}25(0{,}60)=0{,}45\] \[L(0)=0{,}75(0{,}68)+0{,}25(0{,}32)=0{,}59\] \[L(2)=0{,}75(0{,}97)+0{,}25(0{,}03)=0{,}735\]

O posterior inclínase cara a «dominado», pero menos que con \(s=1\).

10. Estimación visible de éxito

A porcentaxe mostrada ao alumno é a probabilidade esperada de acertar ben un exercicio medio:

\[\widehat{P}_t(A) = \sum_i P_t(\theta_i)\cdot P(A\mid \theta_i,b=0)\]

Exemplo con \(P_t(\theta)=(0{,}20,\ 0{,}30,\ 0{,}50)\):

\[\widehat{P}_t(A) = 0{,}20(0{,}40)+0{,}30(0{,}68)+0{,}50(0{,}97) = 0{,}08+0{,}204+0{,}485 = 0{,}769 \approx 77\%\]

11. Entropía de Shannon

A incerteza dunha distribución mídese coa entropía de Shannon:

\[H(P) = -\sum_i p_i\log_2(p_i)\]

Para unha distribución uniforme \((1/3,1/3,1/3)\), a entropía é máxima: \(H=\log_2(3)\). O sistema usa a entropía normalizada:

\[H_{\text{norm}}(P) = \frac{H(P)}{\log_2(3)} \in [0,1]\]

Se \(H_{\text{norm}}=1\), a incerteza é máxima. Se \(H_{\text{norm}}\approx0\), o sistema ten moita confianza.

12. Ganancia de información esperada

Para elixir un exercicio, o sistema calcula canta incerteza espera reducir:

\[IG = H(P) - \bigl[P(A)\,H(P_{\text{post},A}) + P(F)\,H(P_{\text{post},F})\bigr]\]

onde:

\[P(A) = \sum_i P(\theta_i)P(A\mid\theta_i,b)\qquad P(F)=1-P(A)\] \[P_{\text{post},A}(\theta_i) = \frac{P(\theta_i)P(A\mid\theta_i,b)}{P(A)}\qquad P_{\text{post},F}(\theta_i) = \frac{P(\theta_i)P(F\mid\theta_i,b)}{P(F)}\]

13. Intuición da ganancia de información

Un exercicio moi fácil tende a ter unha ganancia de información baixa: se o alumno o acierta, aprendemos pouco porque o acerto era esperable. Un exercicio moi difícil tamén pode ter unha ganancia baixa: se o alumno falla, aprendemos pouco porque o fallo era esperable. Os exercicios máis informativos son os de dificultade próxima ao nivel do alumno, onde ambos os resultados son posibles.

14. Axuste de dificultade

Na fase de reforzo, o sistema tamén valora se a dificultade do exercicio se axusta ao nivel estimado do alumno:

\[Axuste(b,\bar{\theta}) = e^{-0{,}5(b-\bar{\theta})^2}\]

Se \(b=\bar{\theta}\), o axuste é máximo (\(Axuste=1\)). A medida que \(b\) se afasta de \(\bar{\theta}\), o axuste decrece.

15. Puntuación dun exercicio candidato

Durante a fase diagnóstica, o sistema prioriza a ganancia de información:

\[Puntuación_{\text{diagnóstico}}(p)=IG(p)\]

Durante a fase de reforzo, combina ganancia de información e axuste de dificultade:

\[Puntuación_{\text{reforzo}}(p) = 0{,}65\cdot IG_{\text{norm}}(p) + 0{,}35\cdot Axuste(p)\]

16. Selección do tipo de problema

Fase diagnóstica

\[D_t = pendentes_t + H_{\text{norm}}(P_t)\]

onde \(pendentes_t = \max(0, 2-n_t)\) é o número de intentos que aínda faltan para chegar ao mínimo de dous.

Fase de reforzo

\[D_t = 1-\widehat{P}_t(A)\]

O sistema elixe entre os tipos con maior debilidade, introducindo algo de aleatoriedade entre candidatos próximos.

17. Diagnóstico por habilidades transversais

Ademais das distribucións por tipo, o sistema mantén distribucións bayesianas para seis habilidades transversais, con chan de azar propio para cada unha:

18. Por que as porcentaxes de habilidades non son directamente comparables

O chan de azar para «orde» é \(1/2\) e para «tipo» é \(1/6\). Isto significa que o mesmo nivel \(\theta\) produce porcentaxes visibles distintas segundo a habilidade. Un 75 % en orde non ten exactamente o mesmo significado que un 75 % en tipo. O nivel latente \(\theta\) é a referencia común.

19. Gardado do diagnóstico inicial

Cando todos os tipos completaron polo menos dous exercicios, o sistema garda unha copia do estado:

\[Foto = \{P_t(\theta)\}_{t\in T}\]

Isto permite comparar o nivel ao rematar o diagnóstico co nivel actual despois da fase de reforzo:

\[\Delta_t = \widehat{P}_t^{\text{actual}}(A) - \widehat{P}_t^{\text{diagnóstico}}(A)\]

20. Resumo formal do ciclo completo

Paso 1: estado previo: \(P_t^{\text{antigo}}(\theta)\) para cada tipo; \(P_d^{\text{antigo}}(\theta)\) para cada habilidade.

Paso 2: selección do tipo. En diagnóstico: \(t^*=\arg\max_t(pendentes_t + H_{\text{norm}}(P_t))\). En reforzo: \(t^*=\arg\max_t(1-\widehat{P}_t(A))\).

Paso 3: selección do exercicio. En diagnóstico: \(p^*=\arg\max_p IG(p)\). En reforzo: \(p^*=\arg\max_p(0{,}65\,IG_{\text{norm}}(p)+0{,}35\,Axuste(p))\).

Paso 4: o alumno produce sinais \(x_{\text{orde}}, x_{\text{alcance}}, x_{\text{rep}}, x_{\text{tipo}}, x_n, x_k, x_{\text{fórmula}}\).

Paso 5: cálculo da puntuación: \(s = 0{,}10x_{\text{orde}} + 0{,}10x_{\text{alcance}} + 0{,}10x_{\text{rep}} + 0{,}25x_{\text{tipo}} + 0{,}15x_n + 0{,}15x_k + 0{,}15x_{\text{fórmula}}\).

Paso 6: actualización do tipo con \(L(\theta_i)=s\,P(A\mid\theta_i,b)+(1-s)\,P(F\mid\theta_i,b)\).

Paso 7: actualización das habilidades coa regra de Bayes e o sinal binario de cada habilidade.

Paso 8: estimación visible: \(\widehat{P}_t(A) = \sum_i P_t(\theta_i)P(A\mid\theta_i,0)\).

21. Idea matemática central

\[\boxed{\text{Bayes} + \text{IRT} + \text{Shannon} + \text{puntuación parcial}}\]

Bayes actualiza a crenza: \(P(\theta\mid R)\). IRT estima a probabilidade de acerto dado nivel e dificultade: \(P(A\mid\theta,b)\). Shannon mide a incerteza: \(H(P)\). A ganancia de información decide que pregunta reduce máis esa incerteza: \(IG=H_{\text{antes}}-E[H_{\text{despois}}]\). A puntuación parcial transforma unha resposta complexa nun sinal gradual: \(s\in[0,1]\).

22.4 Xeración de exercicios e asignación de dificultade

Cando se esgotan os exercicios dispoñibles para un tipo no banco, o sistema xéraos de forma procedimental. A dificultade \(b\in\{-1,0,1\}\) asígnase mediante regras baseadas no tamaño de \(n\) e \(k\). Estas regras son unha heurística de deseño, non unha calibración psicométrica.

22.5 Diagnóstico de \(n\) e \(k\) como habilidade única

Na puntuación, \(n\) e \(k\) son dous sinais independentes con peso 0,15 cada un. No diagnóstico por habilidades, agrúpanse na habilidade única \(nk\), correcta só se ambos son correctos:

\[r_{nk} = x_n\cdot x_k\]

23. Validación e fiabilidade do diagnóstico (opcional)

As comprobacións desta sección non forman parte do fluxo obrigatorio: son unha capa opcional que aumenta a honestidade do diagnóstico sen necesidade de datos empíricos.

23.1 Axuste do patrón individual (person-fit \(l_z\))

Ao pechar o diagnóstico inicial, o sistema avalía se o patrón de respostas é coherente co nivel estimado. Para cada pregunta respondida durante o diagnóstico, con dificultade \(b_i\) e puntuación observada \(s_i\in[0,1]\), tómase \(\widehat{\theta}\) como a hipótese máis probable (MAP) do tipo correspondente e \(P_i = P(A\mid\widehat{\theta},\,b_i)\). O índice estandarizado de Drasgow é:

\[l_z = \frac{l_0 - E[l_0]}{\sqrt{\operatorname{Var}[l_0]}}\]

con:

\[l_0 = \sum_i \bigl[s_i\ln P_i + (1-s_i)\ln(1-P_i)\bigr]\] \[E[l_0] = \sum_i \bigl[P_i\ln P_i + (1-P_i)\ln(1-P_i)\bigr]\] \[\operatorname{Var}[l_0] = \sum_i P_i(1-P_i)\left[\ln\frac{P_i}{1-P_i}\right]^2\]

Un valor \(l_z\approx0\) indica bo axuste. Un valor moi negativo (orientativamente \(l_z<-2\)) sinala respostas incoherentes coa dificultade —acertos no difícil e fallos no fácil, ou azar—, e o sistema marca o diagnóstico como provisional no informe. É un sinal de cautela, non unha proba formal; con poucas preguntas só é orientativo, polo que se calcula unicamente a partir de tres respostas.

23.2 Separabilidade do deseño (Monte Carlo)

Como propiedade do test (non do alumno), pódese estimar con que fiabilidade o banco distingue os niveis. Xéranse respondentes sintéticos situados no \(\theta\) de cada hipótese, pásaselles o test reutilizando a mesma selección adaptativa e constrúese a matriz de confusión (nivel real fronte a nivel diagnosticado). A diagonal recolle os acertos do deseño e a súa media é a exactitude.

Hai que presentala como fiabilidade baixo o modelo, nunca como validez empírica: os respondentes saen do propio modelo, así que mide se o deseño discrimina os niveis, non se os parámetros reflicten a realidade. É unha ferramenta do creador do recurso, non do alumnado, e non se mostra na interface do alumno: vive nunha utilidade á parte (validacion-banco.html) que o autor pode executar ao deseñar ou revisar o banco.
