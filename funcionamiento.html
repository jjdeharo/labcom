Explicación conceptual

Funcionamiento bayesiano del sistema adaptativo de práctica

Este programa no se limita a contar aciertos y errores. Su funcionamiento se basa en un modelo adaptativo formativo que estima, de manera probabilística, qué tipos de problemas domina el alumno, cuáles está aprendiendo y cuáles necesita reforzar. Para ello combina tres ideas principales: inferencia bayesiana, una función tipo IRT para relacionar nivel y dificultad, y entropía de Shannon para elegir preguntas informativas.

No pretende emitir una calificación psicométrica exacta, sino orientar la práctica: decidir qué conviene preguntar, interpretar los errores y ofrecer un diagnóstico útil.

1. El programa no guarda una nota: guarda una creencia bayesiana

Lo primero que hay que entender es que el sistema no calcula el nivel del alumno como un simple porcentaje de aciertos. No funciona diciendo: «ha acertado 7 de 10, por tanto sabe un 70 %». Hace algo diferente: mantiene una creencia probabilística sobre el estado del alumno.

Para cada tipo de problema, el programa considera tres hipótesis posibles:

HipótesisValor θSignificado
No aprendido−2El alumno todavía no domina ese tipo
Aprendiendo0El alumno está en proceso
Dominado+2El alumno lo resuelve con soltura

Al principio, como el programa no sabe nada del alumno, reparte la probabilidad por igual:

\[P(\text{no aprendido})=\frac13\] \[P(\text{aprendiendo})=\frac13\] \[P(\text{dominado})=\frac13\]

Es decir, para cada tipo empieza así:

\[[1/3,\ 1/3,\ 1/3]\]

Eso no significa que el alumno tenga una nota del 33 %. Significa que, con la información inicial disponible, las tres hipótesis son igual de posibles.

Además, esta creencia se guarda por separado para cada tipo de problema. El alumno puede estar muy avanzado en variaciones con repetición y, al mismo tiempo, tener dificultades en combinaciones. El programa no describe al alumno «en bloque», sino competencia por competencia.

Después, esa distribución se resume mediante un nivel medio:

\[\theta_{\text{estimada}} = P(\text{no aprendido})\cdot(-2) + P(\text{aprendiendo})\cdot0 + P(\text{dominado})\cdot2\]

Por ejemplo, si después de varios ejercicios la creencia sobre combinaciones fuese:

\[[0{,}20,\ 0{,}30,\ 0{,}50]\]

el nivel estimado sería:

\[\theta = 0{,}20\cdot(-2)+0{,}30\cdot0+0{,}50\cdot2=0{,}6\]

Ese 0,6 no es una nota. Es una posición aproximada en una escala interna de dominio.

La idea fundamental es que el «nivel» no es un contador de aciertos, sino una distribución de probabilidad sobre tres estados: no aprendido, aprendiendo y dominado.

2. Bayes entra cuando el alumno responde

Cada vez que el alumno resuelve un ejercicio, el programa actualiza su creencia mediante la regla de Bayes.

La estructura es esta:

\[P(H_i \mid R) = \frac{P(H_i)\cdot P(R \mid H_i)}{\sum_j P(H_j)\cdot P(R \mid H_j)}\]

Donde:

  • \(H_i\) es una hipótesis: no aprendido, aprendiendo o dominado.
  • \(R\) es el resultado observado.
  • \(P(H_i)\) es la creencia antes del ejercicio.
  • \(P(R \mid H_i)\) es la probabilidad de obtener ese resultado si esa hipótesis fuera cierta.

Dicho de forma sencilla, el programa se pregunta:

Si el alumno estuviera realmente en el nivel «no aprendido», ¿qué probabilidad habría de que respondiera así?
Si estuviera en «aprendiendo», ¿qué probabilidad habría?
Si estuviera en «dominado», ¿qué probabilidad habría?

Después compara esas probabilidades y desplaza la creencia hacia la hipótesis que mejor explica la respuesta.

Si el alumno acierta bien un ejercicio, la hipótesis «dominado» gana peso. Si falla, gana peso «no aprendido». Si la respuesta es intermedia, la actualización es más suave.

Este punto es esencial: Bayes no convierte automáticamente aciertos en porcentajes. Lo que hace es mover probabilidad entre hipótesis según lo compatible que ha sido la respuesta del alumno con cada una de ellas.

3. La parte IRT: nivel del alumno frente a dificultad del ejercicio

Para aplicar Bayes, el programa necesita estimar qué probabilidad tendría un alumno de cierto nivel de acertar un ejercicio de cierta dificultad. Ahí entra la parte tipo IRT.

El programa usa una función logística:

\[P(\text{acierto}) = c + \frac{1-c}{1+e^{-a(\theta-b)}}\]

Donde:

  • \(\theta\) es el nivel hipotético del alumno.
  • \(b\) es la dificultad del ejercicio.
  • \(a\) controla la discriminación de la pregunta.
  • \(c\) es el suelo de azar.

La idea es simple: si \(\theta > b\), el nivel del alumno está por encima de la dificultad del ejercicio y el acierto es bastante probable. Si \(\theta < b\), la dificultad supera el nivel estimado y el acierto es menos probable.

Los ejercicios tienen una dificultad aproximada:

DificultadInterpretación
−1Fácil
0Media
+1Difícil

Así, acertar una pregunta difícil aporta más información positiva que acertar una fácil, porque el acierto era menos esperable. Del mismo modo, fallar una pregunta fácil es más significativo que fallar una difícil.

Si un alumno acierta un ejercicio de dificultad media, el programa compara ese acierto con las tres hipótesis \(\theta=-2\), \(\theta=0\) y \(\theta=2\). Si el acierto es mucho más compatible con \(\theta=2\) que con \(\theta=-2\), la probabilidad de «dominado» aumenta.

Dado este nivel hipotético y esta dificultad, ¿qué probabilidad había de acertar? — Bayes usa esa respuesta para actualizar la creencia.

4. Antes de aplicar Bayes, el programa calcula una puntuación parcial

El programa no evalúa un ejercicio solo como «bien» o «mal». Lo divide en varias partes, porque resolver un problema de combinatoria implica varios pasos.

El alumno debe: (1) decidir si importa el orden; (2) decidir si se usan todos los elementos o solo algunos; (3) decidir si hay repetición; (4) nombrar el tipo de problema; (5) identificar \(n\); (6) identificar \(k\); (7) elegir la fórmula.

Cada parte tiene un peso:

Parte evaluadaPeso
Orden10 %
Alcance: todos o algunos10 %
Repetición10 %
Tipo de problema25 %
Identificación de \(n\)15 %
Identificación de \(k\)15 %
Fórmula15 %
\[\text{score} = 0{,}10\cdot\text{orden} + 0{,}10\cdot\text{alcance} + 0{,}10\cdot\text{rep} + 0{,}25\cdot\text{tipo} + 0{,}15\cdot n + 0{,}15\cdot k + 0{,}15\cdot\text{fórmula}\]

Cada parte vale 1 si está bien y 0 si está mal. Por ejemplo, si el alumno acierta el orden, la repetición, el tipo, \(n\) y la fórmula, pero falla el alcance y \(k\):

\[0{,}10 + 0{,}10 + 0{,}25 + 0{,}15 + 0{,}15 = 0{,}75\]

Esto es importante porque el sistema no premia únicamente el resultado final. Lo que cuenta es el proceso: leer el enunciado, clasificar, identificar \(n\) y \(k\), y elegir la fórmula.

5. Qué hace un score de 1, 0,75, 0,5, 0,25 o 0

El score controla la dirección y la intensidad de la actualización bayesiana. Si el score es 1, el ejercicio se interpreta como acierto completo; si es 0, como fallo completo. Valores intermedios producen actualizaciones más suaves.

Para conectar el score con Bayes, el programa necesita saber cuánto respalda ese resultado cada hipótesis. Esa medida se llama verosimilitud, \(L(\theta)\): indica, para cada valor hipotético de θ, cuán compatible es el resultado obtenido con él. Es el término \(P(R\mid H_i)\) de la regla de Bayes, pero expresado a partir del score en lugar de usar solo «acierto» o «fallo».

El programa calcula esta verosimilitud suave así:

\[L(\theta)=\text{score}\cdot P(A\mid\theta)+(1-\text{score})\cdot P(F\mid\theta)\]

donde \(P(F\mid\theta)=1-P(A\mid\theta)\). Esto permite tratar los ejercicios parcialmente correctos de forma gradual.

Si el score es 0,5, la verosimilitud queda prácticamente igual para todas las hipótesis: el sistema no tiene motivos para cambiar la distribución.

Por encima de 0,5, el resultado empuja hacia «dominado».
Por debajo de 0,5, empuja hacia «no aprendido».
En torno a 0,5, apenas informa.

6. Cómo convierte la creencia en porcentaje y semáforo

Después de actualizar Bayes, el programa tiene para cada tipo una distribución como \([0{,}20,\ 0{,}30,\ 0{,}50]\). El alumno no ve esa distribución directamente: el programa la convierte en un porcentaje estimado, una barra y un semáforo.

La probabilidad estimada de éxito en un ejercicio de dificultad media es:

\[P(\text{acierto}) = \sum_i P(H_i)\cdot P(\text{acierto}\mid \theta_i,\ b=0)\]
Con la creencia actual, ¿qué probabilidad tendría este alumno de resolver bien un ejercicio medio de este tipo?

El semáforo traduce el nivel en categorías: rojo (dominio bajo), amarillo (en proceso) y verde (dominado). Además, el programa evita declarar «dominado» demasiado pronto: necesita un mínimo de intentos antes de poder mostrar un dominio alto. Así el panel visible es una traducción pedagógica de la creencia bayesiana interna.

7. Cómo elige qué tipo de problema preguntar

El sistema adaptativo funciona en dos fases: diagnóstico y refuerzo.

Fase diagnóstica

Al principio, el programa intenta obtener una muestra mínima de todos los tipos. Como hay seis tipos de problemas y exige dos ejercicios por tipo, el diagnóstico inicial consta de \(6 \times 2 = 12\) ejercicios.

Mientras algún tipo tenga menos de dos ejercicios realizados, el programa prioriza ese tipo. Pero también tiene en cuenta la incertidumbre, medida con entropía de Shannon:

\[H(p)=-\sum p_i\log_2(p_i)\]

Una distribución uniforme \([1/3,\ 1/3,\ 1/3]\) tiene incertidumbre máxima; una distribución concentrada \([0{,}05,\ 0{,}10,\ 0{,}85]\) tiene incertidumbre baja. Durante el diagnóstico, el sistema prioriza tipos poco practicados y con más incertidumbre.

Fase de refuerzo

Cuando todos los tipos tienen al menos dos ejercicios, termina la fase diagnóstica. Entonces el programa cambia de criterio:

¿En qué tipo tiene el alumno menor probabilidad estimada de éxito?

La debilidad es \(1-P(\text{éxito})\). Si en combinaciones el alumno tiene un 45 % de éxito estimado, su debilidad es 0,55; si en permutaciones tiene un 85 %, su debilidad es 0,15. El programa prioriza combinaciones. No elige siempre el tipo con peor puntuación: selecciona aleatoriamente entre los mejores candidatos para evitar una práctica rígida.

8. Cómo escoge el ejercicio concreto dentro del tipo elegido

Una vez elegido el tipo, el sistema calcula para cada ejercicio candidato su ganancia de información esperada:

Si pongo este ejercicio, ¿cuánto espero reducir mi incertidumbre sobre el nivel del alumno?
\[IG = H(\text{antes}) - \bigl[P(A)\cdot H(\text{después de acierto}) + P(F)\cdot H(\text{después de fallo})\bigr]\]

Un buen ejercicio es aquel que, salga como salga, ayuda a aclarar el nivel del alumno. Por eso no siempre convienen las preguntas más fáciles ni las más difíciles: las más informativas son las que están cerca del nivel del alumno, donde el resultado todavía es incierto.

Durante el diagnóstico, el programa prioriza casi exclusivamente la ganancia de información. Durante el refuerzo, combina dos criterios: un 65 % de ganancia de información y un 35 % de ajuste de dificultad. El ajuste de dificultad mide si la dificultad del ejercicio está cerca del nivel estimado del alumno.

9. El diagnóstico por habilidades: saber en qué falla

Además del nivel por tipo de problema, el programa mantiene un diagnóstico por habilidades transversales:

HabilidadSignificado
OrdenSaber si importa el orden
AlcanceSaber si se usan todos o solo algunos elementos
RepeticiónSaber si hay repetición
TipoNombrar el tipo de problema
\(n,k\)Identificar correctamente \(n\) y \(k\)
FórmulaElegir la fórmula correcta

Cada habilidad tiene su propia creencia bayesiana. Esto permite distinguir entre qué tipo necesita practicar (nivel por tipo) y qué parte del proceso le falla (nivel por habilidad).

Por ejemplo, dos alumnos pueden fallar combinaciones por razones distintas: uno puede identificar bien \(n\) y \(k\) pero elegir mal la fórmula; otro puede fallar desde la lectura del enunciado porque no distingue si importa el orden. Ambos tienen dificultades con combinaciones, pero no necesitan la misma ayuda.

Este diagnóstico no guía directamente la selección de ejercicios, pero es muy útil para interpretar los errores y orientar la intervención docente.

10. Cómo se genera el informe

El informe aparece cuando se completa la fase diagnóstica. El programa guarda una «foto» del estado al acabar los 12 ejercicios iniciales y la compara con el nivel actual después de la fase de refuerzo:

TipoDiagnóstico inicialNivel actual
Combinaciones48 %67 %
Variaciones72 %78 %
Permutaciones90 %91 %

El informe muestra: nivel al terminar el diagnóstico, nivel actual, progreso por tipo, puntos fuertes, tipos que necesitan práctica, recomendación personalizada y diagnóstico por habilidades (separando lectura del enunciado y pasos de resolución).

Además, al cerrar el diagnóstico el programa comprueba si las respuestas fueron coherentes con la dificultad. Si detecta un patrón extraño —acertar lo difícil y fallar lo fácil, o responder al azar—, avisa de que el diagnóstico es poco fiable y conviene tomarlo como provisional. Esta comprobación de coherencia (el índice person-fit) se detalla en el apéndice matemático.

11. El ciclo completo

\[\boxed{\text{estado actual} \rightarrow \text{elección adaptativa} \rightarrow \text{respuesta parcial} \rightarrow \text{score} \rightarrow \text{Bayes} \rightarrow \text{nuevo estado} \rightarrow \text{siguiente elección}}\]

Cuando el alumno empieza, el sistema selecciona un ejercicio, el alumno responde paso a paso, el programa calcula el score ponderado y actualiza el modelo bayesiano del tipo correspondiente y las habilidades transversales. La siguiente vez que se pide un ejercicio, el estado interno ha cambiado: si el alumno ha fallado combinaciones, combinaciones ganará prioridad; si ha acertado muy bien variaciones, variaciones perderá urgencia.

Por eso el programa es adaptativo: no sigue una lista fija de ejercicios, sino que cada respuesta modifica la siguiente decisión.

12. Qué partes son sólidas y qué partes son heurísticas

Aspectos sólidos

El sistema no usa una nota simple: mantiene una creencia probabilística por tipo de problema, lo que permite expresar incertidumbre. Evalúa el proceso, no solo el resultado final. Usa la entropía de Shannon para elegir preguntas informativas, no simplemente las más difíciles. Y realiza una fase diagnóstica antes de reforzar, evitando adaptaciones demasiado rápidas basadas en pocos datos.

Aspectos heurísticos

La función IRT —la curva logística \(P(A\mid\theta,b)\) que estima la probabilidad de acierto dado el nivel del alumno \(\theta\) y la dificultad del ejercicio \(b\)— es heurística: sus parámetros no proceden de una calibración empírica con alumnos reales. La dificultad de las preguntas es aproximada. Los pesos del score son razonables pero discutibles. El suelo de azar simplifica: en la práctica, los errores de los alumnos no son completamente aleatorios. El modelo trata los tipos como competencias bastante independientes, cuando en realidad hay conexiones. Y la escala de tres niveles es clara pero gruesa.

Conclusión

El programa combina:

\[\text{Bayes} + \text{IRT heurística} + \text{entropía de Shannon} + \text{puntuación parcial del proceso}\]

Bayes actualiza la creencia sobre el alumno. La IRT estima qué probabilidad de acierto corresponde a cada nivel y dificultad. Shannon mide la incertidumbre y ayuda a elegir preguntas informativas. La puntuación parcial permite valorar el proceso de resolución, no solo el resultado.

Por eso el sistema no dice simplemente «has acertado tantos ejercicios», sino algo más útil: según tus respuestas, estos tipos parecen dominados, estos están en proceso, estos necesitan refuerzo y estos pasos concretos del razonamiento conviene revisarlos. Su valor principal está en orientar la práctica y hacer visible el proceso de aprendizaje.
