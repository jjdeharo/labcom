Mathematical specification

Mathematical appendix: Bayesian mechanism, IRT and Shannon entropy

This appendix describes in more detail the mathematical mechanism of the adaptive system: how it represents the student's level, how it updates its beliefs with Bayes, how it uses an IRT-type function to relate level and difficulty, how it measures uncertainty with Shannon entropy and how it chooses the next question.

1. Representing level as a probability distribution

For each problem type, the system does not store a single level value, but a probability distribution over three hypotheses:

\[\Theta=\{-2,0,2\}\]

where \(\theta=-2\) represents "not learned", \(\theta=0\) represents "learning" and \(\theta=2\) represents "mastered".

For each type \(t\), the student's state is:

\[P_t(\theta)=\bigl(P_t(-2),\ P_t(0),\ P_t(2)\bigr)\]

At the start, with no prior information:

\[P_t(-2)=P_t(0)=P_t(2)=\frac13\]

This is done independently for each problem type. If the types are:

\[T=\{\text{perm},\ \text{permRep},\ \text{var},\ \text{varRep},\ \text{comb},\ \text{combRep}\}\]

the system maintains six separate distributions. The student does not have "a single global level": they have an independent estimate for each problem type.

2. Expected level

From the distribution for each type, the system can calculate a mean level value:

\[E_t[\theta] = \sum_{\theta_i \in \{-2,0,2\}} P_t(\theta_i)\cdot \theta_i = P_t(-2)(-2)+P_t(0)(0)+P_t(2)(2)\]

Example: if \(P_t(\theta)=(0.20,\ 0.30,\ 0.50)\), then:

\[E_t[\theta] = 0.20(-2)+0.30(0)+0.50(2) = -0.40+0+1 = 0.60\]

This value summarises the distribution, but does not completely replace it. The richer information remains in the full distribution.

3. Probability of a correct answer using an IRT-type function

To update the model, the system needs to estimate the probability that a student with a certain level \(\theta\) would answer an exercise of difficulty \(b\) correctly. It uses a logistic function with a guessing floor:

\[P(A\mid \theta,b) = c+\frac{1-c}{1+e^{-a(\theta-b)}}\]

where \(A\) means correct answer, \(a\) is the discrimination parameter (\(a=1.5\)) and \(c\) is the guessing floor.

Exercise difficulties take values \(b=-1\) (easy), \(b=0\) (medium) or \(b=1\) (hard).

4. Interpretation of the IRT function

The decisive quantity is \(\theta-b\). If \(\theta>b\), the hypothetical level is above the difficulty and the probability of a correct answer is high. If \(\theta=b\), the basic logistic equals \(1/2\), so \(P(A\mid\theta=b)=c+\frac{1-c}{2}\). If \(\theta<b\), the probability falls, but not below \(c\).

Hypothesis\(\theta\)Relation to \(b=0\)Interpretation
Not learned−2\(\theta<b\)Correct answer unlikely
Learning0\(\theta=b\)Intermediate probability
Mastered2\(\theta>b\)Correct answer very likely

That is why a correct answer on a hard question shifts the belief towards "mastered" more than a correct answer on an easy one.

The guessing floor for the global type domain is calculated from the expected chance performance across the whole process:

6. Partial exercise score

Each exercise produces a partial score \(s\in[0,1]\), obtained by weighting seven signals:

SignalWeight
Correct order \((x_{\text{order}})\)0.10
Correct scope \((x_{\text{scope}})\)0.10
Correct repetition \((x_{\text{rep}})\)0.10
Correct type \((x_{\text{type}})\)0.25
Correct \(n\) \((x_n)\)0.15
Correct \(k\) \((x_k)\)0.15
Correct formula \((x_{\text{formula}})\)0.15
\[s = 0.10x_{\text{order}} + 0.10x_{\text{scope}} + 0.10x_{\text{rep}} + 0.25x_{\text{type}} + 0.15x_n + 0.15x_k + 0.15x_{\text{formula}}\]

Each variable equals 1 if the answer is correct and 0 if incorrect. The weights sum to 1, so \(0\leq s\leq1\).

7. Soft likelihood from the score

The exercise result is not just correct or incorrect, but a partial score \(s\). The system therefore uses an interpolated likelihood:

\[L(\theta) = s\cdot P(A\mid \theta,b) + (1-s)\cdot P(F\mid \theta,b)\]

Since \(P(F\mid\theta,b)=1-P(A\mid\theta,b)\), writing \(p_\theta=P(A\mid\theta,b)\):

\[L(\theta) = s\,p_\theta + (1-s)(1-p_\theta)\]

Special cases:

  • If \(s=1\): \(L(\theta)=P(A\mid\theta,b)\) — complete correct answer.
  • If \(s=0\): \(L(\theta)=P(F\mid\theta,b)\) — complete failure.
  • If \(s=0.5\): \(L(\theta)=0.5\) for any \(\theta\) — the Bayesian update is practically zero.

8. Bayesian update

Before the exercise, the state of type \(t\) is \(P_t^{\text{old}}(\theta_i)\). For each hypothesis the likelihood \(L(\theta_i)\) is calculated, then the unnormalised value \(q_i=P_t^{\text{old}}(\theta_i)\cdot L(\theta_i)\), and finally it is normalised:

\[P_t^{\text{new}}(\theta_i) = \frac{P_t^{\text{old}}(\theta_i)\cdot L(\theta_i)}{\sum_j P_t^{\text{old}}(\theta_j)\cdot L(\theta_j)}\]

This is Bayes' rule applied to the three discrete hypotheses.

9. Complete update example

Medium-difficulty exercise (\(b=0\)) with uniform initial distribution \(P^{\text{old}}=(1/3,1/3,1/3)\). Approximate IRT correct-answer probabilities: \(P(A\mid -2,0)\approx0.40\), \(P(A\mid 0,0)\approx0.68\), \(P(A\mid 2,0)\approx0.97\).

Case 1: complete correct answer (\(s=1\))

Likelihoods: \(L(-2)=0.40\), \(L(0)=0.68\), \(L(2)=0.97\). Posterior:

\[P^{\text{new}}(-2)\approx0.195\quad P^{\text{new}}(0)\approx0.332\quad P^{\text{new}}(2)\approx0.473\]

The "mastered" hypothesis gains weight.

Case 2: complete failure (\(s=0\))

Failure likelihoods: \(L(-2)=0.60\), \(L(0)=0.32\), \(L(2)=0.03\). Posterior:

\[P^{\text{new}}(-2)\approx0.632\quad P^{\text{new}}(0)\approx0.337\quad P^{\text{new}}(2)\approx0.032\]

The "not learned" hypothesis gains weight.

Case 3: partial result (\(s=0.75\))

\[L(-2)=0.75(0.40)+0.25(0.60)=0.45\] \[L(0)=0.75(0.68)+0.25(0.32)=0.59\] \[L(2)=0.75(0.97)+0.25(0.03)=0.735\]

The posterior shifts towards "mastered", but less than with \(s=1\).

10. Visible success estimate

The percentage shown to the student is the expected probability of correctly solving a medium exercise:

\[\widehat{P}_t(A) = \sum_i P_t(\theta_i)\cdot P(A\mid \theta_i,b=0)\]

Example with \(P_t(\theta)=(0.20,\ 0.30,\ 0.50)\):

\[\widehat{P}_t(A) = 0.20(0.40)+0.30(0.68)+0.50(0.97) = 0.08+0.204+0.485 = 0.769 \approx 77\%\]

11. Shannon entropy

The uncertainty of a distribution is measured with Shannon entropy:

\[H(P) = -\sum_i p_i\log_2(p_i)\]

For a uniform distribution \((1/3,1/3,1/3)\), entropy is maximum: \(H=\log_2(3)\). The system uses normalised entropy:

\[H_{\text{norm}}(P) = \frac{H(P)}{\log_2(3)} \in [0,1]\]

If \(H_{\text{norm}}=1\), uncertainty is maximum. If \(H_{\text{norm}}\approx0\), the system is very confident.

12. Expected information gain

To choose an exercise, the system calculates how much uncertainty it expects to reduce:

\[IG = H(P) - \bigl[P(A)\,H(P_{\text{post},A}) + P(F)\,H(P_{\text{post},F})\bigr]\]

where:

\[P(A) = \sum_i P(\theta_i)P(A\mid\theta_i,b)\qquad P(F)=1-P(A)\] \[P_{\text{post},A}(\theta_i) = \frac{P(\theta_i)P(A\mid\theta_i,b)}{P(A)}\qquad P_{\text{post},F}(\theta_i) = \frac{P(\theta_i)P(F\mid\theta_i,b)}{P(F)}\]

An exercise has high information gain if, when answered, it greatly reduces the uncertainty about the student's distribution.

13. Intuition behind information gain

A very easy exercise tends to have low information gain: if the student gets it right, we learn little because the correct answer was expected. A very hard exercise can also have low gain: if the student fails, we learn little because the failure was expected.

The most informative exercises tend to be those whose difficulty is close to the student's estimated level, because there both correct and incorrect answers are possible and both scenarios help differentiate hypotheses.

14. Difficulty fit

In the reinforcement phase, the system also values whether the exercise difficulty matches the student's estimated level. It first calculates the expected level \(\bar{\theta}=E[\theta]\), then the fit:

\[Fit(b,\bar{\theta}) = e^{-0.5(b-\bar{\theta})^2}\]

If \(b=\bar{\theta}\), the fit is maximum (\(Fit=1\)). As \(b\) moves away from \(\bar{\theta}\), the fit decreases:

15. Score of a candidate exercise

During the diagnostic phase, the system prioritises information gain:

\[Score_{\text{diagnostic}}(p)=IG(p)\]

During the reinforcement phase, it combines information gain and difficulty fit:

\[Score_{\text{reinforcement}}(p) = 0.65\cdot IG_{\text{norm}}(p) + 0.35\cdot Fit(p)\]

where \(IG_{\text{norm}}(p)=IG(p)/IG_{\max}\) normalises the gain relative to the best available candidate.

16. Problem type selection

Diagnostic phase

The system prioritises types with the highest diagnostic value:

\[D_t = missing_t + H_{\text{norm}}(P_t)\]

where \(missing_t = \max(0, 2-n_t)\) is the number of attempts still needed to reach the minimum of two. This combines lack of sample and uncertainty about the level.

Reinforcement phase

When all types have enough attempts, the system prioritises the weakest types:

\[W_t = 1-\widehat{P}_t(A)\]

The system chooses among types with the highest weakness, introducing some randomness among close candidates to avoid excessive repetition.

17. Transversal skills diagnosis

In addition to the distributions by type, the system maintains Bayesian distributions for six cross-cutting skills:

\[D = \{\text{order},\ \text{scope},\ \text{repetition},\ \text{type},\ \text{nk},\ \text{formula}\}\]

For each skill \(d\), there is a distribution \(P_d(\theta)\) updated with a binary signal \(r_d\in\{0,1\}\). Each skill has its own guessing floor:

SkillGuessing floor
Order\(1/2\)
Scope\(1/2\)
Repetition\(1/2\)
Type\(1/6\)
\(n,k\)\(1/2\)
Formula\(1/6\)
\[P(A_d\mid\theta,b) = c_d + \frac{1-c_d}{1+e^{-a(\theta-b)}}\]

18. Why skill percentages are not directly comparable

The guessing floor for "order" is \(1/2\) and for "type" is \(1/6\). This means that the same level \(\theta\) produces different visible percentages depending on the skill: for a yes/no question the floor is already 50%, while choosing among six types has a floor of ~16.7%.

Therefore a 75% in order does not have exactly the same meaning as a 75% in type. The percentage depends on the guessing floor. The latent level \(\theta\) is the common reference.

19. Saving the initial diagnosis

When all types have at least two exercises completed, the system saves a copy of the state:

\[Snapshot = \{P_t(\theta)\}_{t\in T}\]

This snapshot allows comparing the level at the end of diagnosis with the current level after the reinforcement phase:

\[\Delta_t = \widehat{P}_t^{\text{current}}(A) - \widehat{P}_t^{\text{diagnosis}}(A)\]

If \(\Delta_t>0\) there is estimated improvement; if \(\Delta_t<0\) there is decline; if \(\Delta_t\approx0\) the level is stable.

20. Formal summary of the complete cycle

Step 1: prior state

For each type: \(P_t^{\text{old}}(\theta)\). For each skill: \(P_d^{\text{old}}(\theta)\).

Step 2: type selection

In diagnostic: \(t^*=\arg\max_t\bigl(missing_t + H_{\text{norm}}(P_t)\bigr)\). In reinforcement: \(t^*=\arg\max_t\bigl(1-\widehat{P}_t(A)\bigr)\), with random selection among candidates close to the maximum.

Step 3: exercise selection

In diagnostic: \(p^*=\arg\max_p IG(p)\). In reinforcement: \(p^*=\arg\max_p\bigl(0.65\,IG_{\text{norm}}(p)+0.35\,Fit(p)\bigr)\).

Step 4: student's response

The student produces signals: \(x_{\text{order}}, x_{\text{scope}}, x_{\text{rep}}, x_{\text{type}}, x_n, x_k, x_{\text{formula}}\).

Step 5: score calculation

\[s = 0.10x_{\text{order}} + 0.10x_{\text{scope}} + 0.10x_{\text{rep}} + 0.25x_{\text{type}} + 0.15x_n + 0.15x_k + 0.15x_{\text{formula}}\]

Step 6: type update

\[P_{t^*}^{\text{new}}(\theta_i) = \frac{P_{t^*}^{\text{old}}(\theta_i)\,L(\theta_i)}{\sum_j P_{t^*}^{\text{old}}(\theta_j)\,L(\theta_j)} \qquad L(\theta_i)=s\,P(A\mid\theta_i,b)+(1-s)\,P(F\mid\theta_i,b)\]

Step 7: skills update

For each skill \(d\): same Bayesian rule with \(L_d\) using the binary result for that skill and its guessing floor \(c_d\).

Step 8: visible estimate

\[\widehat{P}_t(A) = \sum_i P_t(\theta_i)P(A\mid\theta_i,0) \qquad E_t[\theta] = \sum_i P_t(\theta_i)\,\theta_i\]

These values generate bars, traffic light and report.

21. Central mathematical idea

\[\boxed{\text{Bayes} + \text{IRT} + \text{Shannon} + \text{partial score}}\]

Bayes updates the belief: \(P(\theta\mid R)\). IRT estimates the probability of a correct answer given level and difficulty: \(P(A\mid\theta,b)\). Shannon measures uncertainty: \(H(P)\). Information gain decides which question reduces that uncertainty the most: \(IG=H_{\text{before}}-E[H_{\text{after}}]\). The partial score transforms a complex response into a gradual signal: \(s\in[0,1]\).

The result is a system in which each response modifies the student's internal state, and that new state modifies the next question.

Implementation details

22. Implementation details

The preceding sections describe the mathematical mechanism. This section records the concrete implementation decisions that affect what the student sees, but are not part of the Bayesian core.

22.1 Caps on the visible percentage based on number of attempts

The raw estimate \(\widehat{P}_t(A)\) is not always shown directly. To avoid overoptimistic conclusions with little evidence, the system caps the visible percentage based on the number of attempts \(n_t\). The thresholds are derived from the IRT function at medium difficulty:

\[\text{LOW\_MAX} = \operatorname{round}\bigl(100\cdot P(A\mid\theta=-0.3,\,b=0)\bigr) = 61\] \[\text{HIGH\_MIN} = \operatorname{round}\bigl(100\cdot P(A\mid\theta=1.4,\,b=0)\bigr) = 93\]

The displayed percentage is:

\[\widehat{P}_t^{\text{visible}} = \begin{cases} \min\bigl(\widehat{P}_t(A),\ 60\bigr) & \text{if } n_t < 2 \\ \min\bigl(\widehat{P}_t(A),\ 92\bigr) & \text{if } 2 \le n_t < 4 \\ \widehat{P}_t(A) & \text{if } n_t \ge 4 \end{cases}\]

22.2 Traffic-light thresholds

The traffic-light colour depends on the expected level \(\bar{\theta}_t\) and the number of attempts \(n_t\):

ConditionColour

\(n_t < 2\)Red
\(2\le n_t < 4\) and \(\bar{\theta}_t\ge-0.3\)Yellow
\(2\le n_t < 4\) and \(\bar{\theta}_t<-0.3\)Red

The cut-offs \(\bar{\theta}=-0.3\) and \(\bar{\theta}=1.4\) are the same ones that generate the LOW_MAX and HIGH_MIN bands in section 22.1, so colour and percentage are consistent with each other.

22.3 Randomness in selection

A candidate \(p\) enters the eligible set if \(Score(p)\ge Score_{\max}-\tau\), with different tolerances per context:

Type in reinforcement phase0.03
Exercise in diagnostic phase\(\max(0.10\cdot Score_{\max},\ \varepsilon)\)

22.4 Exercise generation and difficulty assignment

When the available exercises for a type in the bank are exhausted, the system generates new exercises procedurally. The difficulty \(b\in\{-1,0,1\}\) is assigned using rules based on the size of \(n\) and \(k\) (or the numerical result, depending on the type). These rules are a design heuristic, not a psychometric calibration.

22.5 Diagnosis of \(n\) and \(k\) as a single skill

In the score calculation, \(n\) and \(k\) are two independent signals with weight 0.15 each. In the cross-cutting skills diagnosis, however, both are grouped into the single skill \(nk\), which is considered correct only if \(n\) and \(k\) are both correct:

\[r_{nk} = x_n\cdot x_k\]

The score can grant partial credit for getting only one right, but the cross-cutting skill \(nk\) requires both to be correct.
